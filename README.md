1. Постановка задачи
🔹 Профессиональная проблема
Ручной учёт семейных доходов и расходов (в таблицах, приложениях без аналитики) имеет следующие недостатки:
❌ Тратится 3–5 часов в неделю на категоризацию и свод данных;
❌ Сложно выявить скрытые паттерны трат (сезонность, импульсивные покупки);
❌ Отсутствуют персонализированные, контекстные рекомендации по оптимизации;
❌ Визуализация данных требует навыков работы с графиками/дизайном.
🔹 Цель проекта
Создать прототип веб-сайта для ведения семейного бюджета, который:
Принимает данные о доходах/расходах (текст/таблица);
Автоматически категоризирует траты и находит аномалии с помощью LLM;
Генерирует наглядную аналитику (диаграммы, тренды) через AI-инструмент визуализации;
Формирует 3–5 конкретных, выполнимых советов по финансовой оптимизации.
🔹 Целевая аудитория
Семьи с доходом от 50 000 ₽/мес, ведущие учёт вручную;
Начинающие пользователи, которым важна простота и наглядность;
Люди, желающие повысить финансовую грамотность через персонализированные подсказки.
